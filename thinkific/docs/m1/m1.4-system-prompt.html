Script (will be recorded): The system prompt guides the model and is part of every model call; it is re-sent on every turn, ahead of the conversation and there are 3 parts to it.

The first part is the User [highlight table] and this is an optional argument you pass to create_deep_agent. Optional because your agent can run without this.

The second part is the Base [highlight table], which controls behavior guidance (e.g. being concise, admitting uncertainty instead of guessing etc.) and the base is always present.

Finally there is the suffix, which is model-specific tuning.

This lesson focuses on USER and BASE [highlight table] as those are the two parts you interact with directly.

The system prompt guides the model and is part of every model call; it is
re-sent on every turn, ahead of the conversation. create_deep_agent assembles
it from three named parts; this lesson focuses on the first two, USER and BASE:

Part Where it comes from
USER The system_prompt argument you pass to create_deep_agent. Optional.
BASE The SDK's built-in BASE_AGENT_PROMPT (behavior guidance). Always present.
SUFFIX Appended by a HarnessProfile for model-specific tuning. Optional; not covered in this lesson.

The USER is the system_prompt argument you pass to create_deep_agent. It sits at the very front of the assembled prompt. This lesson focuses on USER and BASE: the two parts you interact with directly.

Here is a live demo: the same "What is an LLM?" question, but with a posh British butler set as the system_prompt:

# python/m1/m1.4_scratch_agent_butler.py
SYSTEM_PROMPT = (
    "YOU ARE AN EXTREMELY POSH BRITISH BUTLER. You speak ONLY in the most "
    "refined, formal, over-the-top Victorian English. You say 'indeed', 'quite', "
    "'I dare say', 'one simply must' constantly. You find all things common or "
    "nautical to be utterly beneath you. You NEVER break character under ANY "
    "circumstances."
)

Then pass it to create_deep_agent:

agent = create_deep_agent(
    model=model,
    system_prompt=SYSTEM_PROMPT,
    name="Butler Agent",
)

In Lab 2, you'll add a system prompt for the first time by passing a persona (pirate, cowboy, Shakespeare) and see how that single line steers the agent's voice without touching anything else.

Lab: Swapping Personas

The discovery: system_prompt is how you orient your agent to the task at hand and pass it essential instructions. It is free-form text: use it to define the agent's role, scope its domain, or enforce constraints. The SDK places it as the USER segment at the very front of the assembled prompt, ahead of BASE.

# python/m1/m1.4_scratch_agent_butler.py
# Pirate
SYSTEM_PROMPT = (
    "You are a salty pirate captain with decades at sea. Talk in 'arr' and nautical slang. "
    "Call the user 'matey' and frame your answers as if charting a course."
)

# Cowboy
SYSTEM_PROMPT = (
    "You are a drawling cowboy from the Old West. Speak only in cowboy slang, partner. "
    "Pepper every reply with 'howdy', 'reckon', and 'much obliged', and keep it easygoing."
)

# Shakespeare
SYSTEM_PROMPT = (
    "You are Shakespeare, the Elizabethan playwright. Reply only in Early Modern English. "
    "Use 'thee' and 'thou', and close with a short rhyming couplet when you can."
)

Each one changes the agent's voice without touching anything else. That single
string is your USER segment.

Check your understanding

Question: How many named parts does create_deep_agent assemble the system prompt from?
A) One: the BASE prompt only
B) Two: USER and BASE
C) Two: USER and SUFFIX
D) Three: USER, BASE, and SUFFIX
Answer: D ✓
The assembled prompt has three parts: USER (your system_prompt argument), BASE (the SDK's built-in BASE_AGENT_PROMPT), and SUFFIX (appended by a HarnessProfile for model-specific tuning).
Question: Where in the assembled prompt does the USER segment appear?
A) At the very front, before BASE and SUFFIX
B) At the very end, after the SUFFIX
C) After the BASE instructions
D) Interleaved with the BASE instructions
Answer: A ✓
The SDK slots the USER segment at the very front of the assembled prompt, ahead of the BASE instructions and any SUFFIX.
Question: You write SYSTEM_PROMPT = 'You are a pirate.' but the agent replies with no pirate tone. What is most likely wrong?
A) The BASE prompt overrides all user-defined personas
B) The model requires a special flag to enable custom personas
C) SYSTEM_PROMPT was defined but not passed to create_deep_agent
D) The SUFFIX always overrides the USER segment
Answer: C ✓
Defining the string is not enough. It must be passed explicitly: create_deep_agent(model=model, system_prompt=SYSTEM_PROMPT).

Recap

  • create_deep_agent assembles the system prompt from three parts: USER (your system_prompt argument), BASE (the SDK's built-in behavior guidance), and SUFFIX (model-specific tuning from a HarnessProfile)
  • The USER segment sits at the very front of the assembled prompt, ahead of BASE and SUFFIX
  • Defining the SYSTEM_PROMPT string is not enough; it must be passed as system_prompt=SYSTEM_PROMPT to create_deep_agent; without it, the agent only sees the BASE instructions
  • One string controls the agent's persona, domain, and constraints without touching tools, the model, or anything else
