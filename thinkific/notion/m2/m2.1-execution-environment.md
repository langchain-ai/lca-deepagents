Deep Agents expose a filesystem surface to the agent: tools like `ls`, `read_file`, `write_file`, `edit_file`, `glob`, and `grep`. These tools don't operate directly on disk. They go through a pluggable **backend** that you configure when you create the agent.

> 📊 [Diagram: backend landscape — Filesystem Tools → Backend → State / Disk / Store / ContextHub / Sandbox / LocalShell / Composite / Custom; Sandbox and LocalShell add `execute` tool — see live lesson]

---

## Recap

- The filesystem **surface** (the tools) is fixed: `ls`, `read_file`, `write_file`, `edit_file`, `glob`, `grep`
- The **backend** is swappable; it controls where files are actually stored
- Sandbox and Local Shell backends add an `execute` tool for running shell commands
- Composite backend routes different paths to different backends
