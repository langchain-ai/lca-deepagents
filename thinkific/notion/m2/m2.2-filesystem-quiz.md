**Q1: What do all filesystem backends have in common?**

A) They all persist files across threads
B) They all require a namespace function
C) They all use the same underlying storage system
D) They all expose the same tool surface (`ls`, `read_file`, `write_file`, etc.); the agent cannot tell which backend it is using

*Answer: D â€” All backends implement the same interface: `ls`, `read_file`, `write_file`, `edit_file`, `glob`, `grep`. The agent code stays the same regardless of which backend you choose â€” swapping the backend is transparent to the agent.*

**Q2: What is the key difference between State and Store backends?**

A) State supports permissions; Store does not
B) State is faster but requires more configuration
C) State is thread-scoped and ephemeral; Store persists files across threads and sessions
D) Store requires a Composite wrapper to function

*Answer: C â€” State keeps files in LangGraph state scoped to the current thread â€” they disappear when the thread ends. Store is backed by a database (PostgreSQL in production) and files persist across threads and sessions.*

**Q3: What makes Context Hub different from other backends?**

A) It stores files in an encrypted database
B) It requires explicit `read_file` calls like other backends
C) Files are injected directly into the model's context window without an explicit read call
D) It is only available in production deployments

*Answer: C â€” Context Hub injects files directly into the model's context window on every call, without requiring an explicit `read_file` call. This is useful for reference material the agent should always have in view.*

**Q4: How are `FilesystemPermission` rules evaluated?**

A) All matching rules are applied in sequence
B) The most specific path pattern wins
C) Rules are evaluated in reverse order; the last match wins
D) Rules are evaluated in order; the first match wins. If nothing matches, the operation is allowed

*Answer: D â€” `FilesystemPermission` rules are evaluated in order. The first rule whose operations and paths match wins. If no rule matches, the operation is allowed by default.*

**Q5: What does `interrupt` mode do in a `FilesystemPermission` rule?**

A) It logs the operation and allows it anyway
B) It raises an exception and halts the agent
C) It pauses for human approval instead of blocking silently
D) It redirects the file operation to a different backend

*Answer: C â€” The `interrupt` mode pauses for human approval instead of blocking silently. This is the same HITL pattern seen with `write_sql` â€” the operation is held until a human approves or cancels it.*
