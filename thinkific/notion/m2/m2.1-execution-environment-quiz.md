# Quiz: Execution Environment

> **Live lesson:** https://langchain-ai.github.io/lca-deepagents/m2/m2.1-execution-environment.html

**Q1: What three capabilities does a Deep Agent's execution environment provide?**

A) Filesystem, database, and interpreter
B) Shell, interpreter, and model
C) Filesystem, shell (execute tool), and interpreter
D) Filesystem, shell, and model

*Answer: C — A Deep Agent's execution environment provides three things: a filesystem to read and write files, a shell to run commands via the `execute` tool, and an interpreter to run code.*

---

**Q2: What do filesystem tools and the `execute` tool both run through?**

A) The agent main loop
B) The LLM directly
C) A swappable backend
D) The interpreter

*Answer: C — Every filesystem and `execute` call passes through whichever backend you configure. The backend is what makes the execution environment configurable — swap it and you change where files live and where commands run.*

---

**Q3: What is the default backend, and where does it keep files?**

A) Filesystem backend; persists files to local disk
B) State backend; keeps files in the agent's in-memory state
C) Store backend; writes to remote storage
D) Context Hub; managed hosted storage

*Answer: B — State is the default backend. Files live in the agent's in-memory state, scoped to the current thread. Nothing persists to disk, which is fine for simple use cases or demos.*

---

**Q4: How is the interpreter different from filesystem tools and the `execute` tool?**

A) It runs through a separate Interpreter backend
B) It is only available in the Sandbox backend
C) It calls the LLM directly instead of executing code
D) It runs as a tool in the agent main loop, not through a backend

*Answer: D — The interpreter is the exception. Unlike filesystem tools and `execute` — which run through a backend — the interpreter runs as a tool in the agent's main loop.*

---

**Q5: Which backend routes file operations to different backends based on path prefix?**

A) Custom
B) Store
C) Context Hub
D) Composite

*Answer: D — Composite routes different path prefixes to different backends. For example, `/memories/` could go to Store while everything else falls back to State — all within the same agent.*
