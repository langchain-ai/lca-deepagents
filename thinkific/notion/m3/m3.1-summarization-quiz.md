# Quiz: Message History & Summarization

> **Live lesson:** https://langchain-ai.github.io/lca-deepagents/m3/m3.1-summarization-quiz.html

**Q1: At what threshold does SummarizationMiddleware fire by default?**

A) 85% of the model's `max_input_tokens`
B) 50% of the model's `max_input_tokens`
C) When the context window is completely full
D) After a fixed count of 20 messages

*Answer: A — For Claude Haiku 4.5 that is 170,000 tokens.*

---

**Q2: How is the summary itself generated when SummarizationMiddleware fires?**

A) By truncating the oldest messages until the token count drops below the limit
B) By a fixed template that extracts the first sentence of each message
C) By a separate LLM call using the same model as the agent
D) By the agent model as part of generating its next response

*Answer: C — The middleware makes its own dedicated LLM call, separate from the agent's response turn.*

---

**Q3: Where in the message list does the generated summary appear?**

A) Appended to the system prompt as an additional instruction block
B) As the very first message in `state["messages"]`
C) As a ToolMessage returned by a built-in summarize tool
D) As a HumanMessage injected at the top of the preserved recent messages

*Answer: D*

---

**Q4: What happens to `state["messages"]` when summarization fires?**

A) Older messages are deleted in place and replaced by the summary message
B) `state["messages"]` is never mutated; summarization only affects what is sent to the model each turn
C) `state["messages"]` is cleared and rebuilt from the summary on every subsequent turn
D) A new summarized state replaces the original, and the old state is discarded

*Answer: B. `state["messages"]` is never modified; the full message history is always preserved intact. This is intentional: summarization is lossy, so keeping the raw history means you can always audit exactly what the agent said and did, replay or inspect a conversation, and access details the summary may have omitted. Summarization only affects what is sent to the model each turn, not what is stored.*

---

**Q5: Why must you pass the model object to `create_deep_agent` when using a profile override, rather than the model ID string?**

A) The string form does not support multimodal inputs required for summarization
B) String initialization disables SummarizationMiddleware entirely
C) Passing the string re-initializes the model internally and discards the profile override
D) The profile attribute only exists on string-initialized models, not objects

*Answer: C*
