# Quiz: Delegation

> **Live lesson:** https://langchain-ai.github.io/lca-deepagents/m4/m4.1-delegation.html

**Q1: What is the job of the Deep Agents planning tool (write_todos)?**

A) It keeps a structured to-do list and tracks each task's status (pending → in progress → completed) as the agent works
B) It runs the tasks in parallel across several models
C) It deletes old messages once the context window is full
D) It writes the final answer to a file

*Answer: A — The planning tool maintains a structured to-do list, persisted in the agent's state, so the agent has an explicit plan to track against on a long, multi-step job.*

---

**Q2: How does a main agent hand work to a subagent in Deep Agents?**

A) By editing the subagent's source code at runtime
B) By forwarding its entire message history to the subagent
C) Subagents cannot be invoked; they run automatically every turn
D) Through a built-in `task` tool — it calls the subagent like any other tool, and can fire off several at once

*Answer: D — Subagents use the subagent-as-tool model: the main agent calls the built-in task tool with an assignment. Because it is just a tool call, the main agent can invoke several subagents in parallel.*

---

**Q3: When the main agent delegates to a subagent, what does the subagent receive and return at the hand-off boundary?**

A) The main agent's full message history; it returns the updated history
B) It shares the main agent's context directly, so nothing is passed
C) One task in; it returns one final result back
D) A task in; it returns its entire internal context

*Answer: C — A subagent runs in an isolated context. It receives one task — not the main agent's history — and returns one final result. Only the result crosses the boundary, keeping the subagent's work and tools out of the main agent's context.*

---

**Q4: What is the difference between a synchronous and an asynchronous subagent?**

A) Synchronous ones run in the background; asynchronous ones block the main agent
B) A synchronous subagent blocks the main agent until it finishes; an asynchronous one runs in the background while the main agent stays responsive
C) Synchronous subagents can use tools; asynchronous ones cannot
D) There is no difference; the terms are interchangeable

*Answer: B — A synchronous subagent blocks the main agent until it returns. An asynchronous subagent runs as a background job so the main agent stays responsive and collects the result later.*
