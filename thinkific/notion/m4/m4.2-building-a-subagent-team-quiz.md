# Quiz: Building a Subagent Team

> **Live lesson:** https://langchain-ai.github.io/lca-deepagents/m4/m4.2-building-a-subagent-team.html

**Q1: In this lab, why is there a single genre-researcher subagent type instead of four separate ones?**

A) Deep Agents only allows one subagent type per main agent
B) All four researchers do the same job — the genre is just the task each one is handed
C) Separate subagents cannot run in parallel
D) It is the only way to share a filesystem between them

*Answer: B — The four researchers are identical in behavior; only their assignment differs. So we define one genre-researcher type and the editor delegates to it once per genre, passing the genre as the task.*

---

**Q2: You set `tools=[internet_search]` on the genre-researcher. What does the subagent end up with?**

A) `internet_search` added on top of all the main agent's tools
B) Every tool except `internet_search`
C) Only `internet_search` (plus the always-available built-in filesystem tools) — the inherited set is replaced
D) No tools at all, because tools cannot be set on a subagent

*Answer: C — Setting tools replaces the inherited set rather than adding to it, so the researcher gets internet_search only. The built-in filesystem tools (write_file, read_file, …) are always provided by the harness, which is why it can still write its notes.*

---

**Q3: The researchers save bulky raw search results to `/research/<genre>/` but return only a short segment. Why return just the segment?**

A) Files are the only way a subagent can return data to the main agent
B) Returning only the distilled segment keeps the raw research out of the editor's context; the file is just scratch space the researcher reads while drafting
C) Because messages cannot contain more than a few words
D) So the editor can rewrite the researchers' notes before publishing

*Answer: B — A subagent returns one message to the editor. Returning only the polished segment — not the bulky raw results — is what keeps the editor's context clean. The private file is simply where the researcher keeps the material it needs while drafting.*

**Q4: The agent runs on the default `StateBackend` and calls `write_file('/output/newsletter.html', …)`. How does the file end up on your disk?**

A) `StateBackend` writes directly to your local filesystem
B) It doesn't — the file is only ever in the agent's chat history
C) The agent requests OS permission to write to disk
D) It doesn't write to disk itself; host code reads the saved file from agent state after invoke and writes it out

*Answer: D — StateBackend is an ephemeral, in-state filesystem — the agent has no real disk access. After invoke returns, trusted host code reads the file from result['files'] and writes it to disk, keeping the agent sandboxed.*
