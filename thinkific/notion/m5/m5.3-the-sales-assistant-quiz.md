# Quiz: The Sales Assistant

> **Live lesson:** https://langchain-ai.github.io/lca-deepagents/m5/m5.3-the-sales-assistant.html

**Q1: Why does `agent.py` use an async `make_graph` factory instead of a module-level `agent = create_deep_agent(...)`?**

A) To make the agent start faster
B) So the agent can be invoked with `asyncio.run()`
C) Because the mail tools come from an MCP server, and getting its tool list is an async call that must finish before the agent is built
D) Because `langgraph dev` cannot load module-level variables

*Answer: C — A model can only call tools it has been told about. The mail tools come from an MCP server whose tool list arrives via an async call, so discovery must be awaited before the agent is assembled — hence an async factory.*

---

**Q2: What is the `--allow-blocking` flag on `langgraph dev` for?**

A) Discovering the MCP server tools does a little synchronous I/O at startup, which the blocking-call detector would otherwise flag
B) It allows the agent to block on human approvals
C) It enables blocking network calls to mail
D) It is required for every `langgraph dev` server

*Answer: A — Launching the mock mail server to list its tools is synchronous I/O at startup. `langgraph dev` normally raises on blocking calls; the flag marks this expected one as allowed.*

---

**Q3: The two human approvals (saving an email draft, adding a customer) are gated on specialist subagents rather than the main agent. Why?**

A) To make the approval cards look nicer
B) Subagents run faster than the main agent
C) It is the only place interrupts are allowed
D) The general-purpose subagent inherits the main agent's tools, so a gated tool on the main agent could be reached ungated; keeping writes on gated specialists makes the design fail-closed

*Answer: D — The always-present general-purpose subagent inherits the main agent's tools. If a write lived on the main agent it could be reached ungated through delegation. Placing both writes on gated specialists means the only path to either write runs through its approval gate.*

**Q4: In this project, what are `AGENTS.md` and the files under `skills/`?**

A) Compiled configuration generated at build time
B) Plain-text files that describe the agent behavior — an operating manual and one playbook per task
C) Python modules that define the subagents
D) Database fixtures for the Chinook data

*Answer: B — AGENTS.md is the operating manual and each skill is a step-by-step playbook — all natural language describing what the agent should do, not code.*
