## From a script to a service

`langgraph dev` (part of the `langgraph-cli` package) provides a cloud-deployment-like environment on your local machine: it hosts your agent behind an HTTP API and runs the supporting infrastructure around it.

> 🖼️ [Diagram: "Studio (chat + debug UI)" connects by HTTP into a dashed "langgraph dev" box containing "Your deep agent" above four built-in service tiles: HTTP API, Persistence, Store, and Task queue — see live lesson]

### Studio: a client for that API

`langgraph dev` includes a chat and debugging interface called "Studio". Studio uses the API endpoints provided by `langgraph dev` to allow you to converse with your agent via chat as well as examine and control the underlying agentic loop via a graphical debugger.

---

## The agent's API

Every agent served by `langgraph dev` exposes the same **Agent Server API**. Once it's up, open **`http://localhost:2024/docs`** in your browser to see the live, interactive API for your agent.

The endpoints group into a handful of categories:

**Assistants** — a configured instance of a graph (graph + its configuration):
- `POST /assistants`, `POST /assistants/search`, `GET /assistants/{id}/graph`, `GET /assistants/{id}/schemas`

**Threads** — the accumulated state of a conversation:
- `POST /threads`, `GET /threads/{id}/state`, `GET /threads/{id}/history`, `POST /threads/search`

**Thread Runs** — one invocation of the agent on a thread:
- `POST /threads/{id}/runs`, `POST /threads/{id}/runs/stream`, `POST /threads/{id}/runs/wait`, `POST /threads/{id}/runs/{run_id}/cancel`

**Store** — long-term key-value memory available from any thread:
- `PUT /store/items`, `GET /store/items`, `POST /store/items/search`, `GET /store/namespaces`

**Advanced** — Stateless Runs, Crons, Streaming, A2A, MCP, System endpoints.

## Beyond local deployment

`langgraph dev` is intended for local development. Deploying an agent to production introduces additional infrastructure considerations. Key resources:

- [**Deployment options**](https://docs.langchain.com/langsmith/deployment): the docs overview of available hosting models (Cloud, standalone server, and self-hosted).
- [**LangSmith Deployment**](https://academy.langchain.com/courses/langsmith-deployment): a LangChain Academy course on deploying, managing, and controlling agents in production.

---

## Recap

- A deployed agent runs as a **service reached over HTTP**, not a script you `invoke()` in Python.
- **`langgraph dev`** hosts your agent and exposes it as an **API**: declared by a small **`langgraph.json`** that points at your graph.
- The **Agent Server API** is the same for every agent: **Assistants**, **Threads**, **Thread Runs**, and **Store**, plus advanced endpoints. The running server documents itself at **`localhost:2024/docs`**.
- **`langgraph dev` ships a client** for that API: **LangGraph Studio**, a combined **chat + debug** interface — and any other chat UI talks to the agent the same way, over HTTP.
