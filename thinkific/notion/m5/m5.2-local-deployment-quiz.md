# Quiz: Local Deployment

> **Live lesson:** https://langchain-ai.github.io/lca-deepagents/m5/m5.2-local-deployment.html

**Q1: What does it mean to deploy an agent with `langgraph dev`?**

A) The agent runs as a service reached over an HTTP API, rather than a script you invoke in Python
B) The agent is compiled into a standalone binary you distribute to users
C) The agent is uploaded to LangSmith Cloud automatically
D) The agent can only be run from inside LangGraph Studio

*Answer: A — A deployed agent runs as a service: langgraph dev hosts it behind an HTTP API, and clients reach it over HTTP instead of calling agent.invoke() in a Python script.*

---

**Q2: What is the role of `langgraph.json`?**

A) It stores the conversation history for each thread
B) It holds your API keys and secrets
C) It declares which graph `langgraph dev` should serve, where its dependencies are, and which env file to load
D) It defines the HTTP endpoints the server exposes

*Answer: C — langgraph.json is the deployment config. It points at your graph (e.g. `./agent.py:graph`), lists dependencies, and names the env file to load. The server derives its endpoints automatically.*

---

**Q3: In the Agent Server API, what is an Assistant?**

A) A single invocation of a graph on a thread
B) The accumulated state of a conversation
C) A long-term key-value memory store
D) A configured instance of a graph — a graph plus its configuration

*Answer: D — An Assistant is a configured instance of a graph: the graph plus the configuration it runs with. A run is one invocation; a thread holds accumulated state; the store is long-term memory.*

---

**Q4: What is LangGraph Studio?**

A) A separate runtime that replaces `langgraph dev` in production
B) A client for the Agent Server API — a combined chat and graphical debugging interface that talks to the agent over HTTP
C) A command-line tool for editing `langgraph.json`
D) A hosted cloud service that deploys your agent

*Answer: B — Studio is a client of the same Agent Server API. It provides both a chat interface and a graphical debugger, and it reaches the agent over HTTP — exactly as any other chat UI would.*
