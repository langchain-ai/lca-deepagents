## MCP: Connecting Agents to External Services

Model Context Protocol (MCP) is an open protocol that standardizes how applications provide tools and context to LLMs. Instead of each integration requiring custom code in your agent, MCP servers expose a consistent interface that any MCP-aware client can connect to.

<iframe src="images/m1.l6-mcp-slides-1.html" style="width:100%;height:460px;border:none;border-radius:12px;display:block;margin:1.25rem 0 1.5rem;" title="MCP architecture walkthrough; use arrow keys or Prev/Next to navigate"></iframe>

LangChain agents connect to MCP servers using the `langchain-mcp-adapters` library. The adapters wrap each MCP tool as a standard LangChain `BaseTool`, so the agent sees it identically to any locally-defined tool. The tool result is added to the message history and the model decides what to do next, exactly as it would with a regular tool call.

### Configuration

deepagents discovers MCP servers from `.mcp.json` in your project root (or `~/.deepagents/.mcp.json` for user-level config). The project repo already includes a config pointing at the LangChain documentation server:

<iframe src="images/m1.l6-mcp-slides-config.html" style="width:100%;height:380px;border:none;border-radius:12px;display:block;margin:1.25rem 0 1.5rem;" title=".mcp.json configuration diagram"></iframe>

Load those tools in Python with `get_mcp_tools`:

```python
from deepagents_code.mcp_tools import get_mcp_tools

tools, session_manager, server_infos = await get_mcp_tools(".mcp.json")
```

Pass `tools` to `create_deep_agent` as you would any other tool list. When you are done, call `await session_manager.cleanup()` to close the connections.

### Transport

The transport is the underlying mechanism used to communicate with an MCP server. It is abstracted away from your agent code; the agent never sees or interacts with it directly.

There are three transport types:

- **stdio** runs the server as a local subprocess on your machine. Messages pass as JSON through the subprocess's standard input and output streams. No networking required.
- **HTTP** sends a standard HTTPS request to a remote server and waits for the response. A new connection opens for each tool call and closes when the response arrives.
- **SSE** (Server-Sent Events) opens a persistent HTTP connection. The server can push events back over that connection in real time, which is useful for long-running tools or streaming results. SSE is the newest transport type in the MCP spec.

<iframe src="images/m1.l6-mcp-slides-2.html" style="width:100%;height:460px;border:none;border-radius:12px;display:block;margin:1.25rem 0 1.5rem;" title="Transport types walkthrough; use arrow keys or Prev/Next to navigate"></iframe>

In `.mcp.json`, set `"type"` to `"http"` or `"sse"` for remote servers, or use `"command"` and `"args"` for stdio servers. deepagents normalizes the transport at load time; your Python code does not change.

### Authorization

Some MCP servers require the user to authenticate before the agent can call their tools.

<iframe src="images/m1.l6-mcp-slides-auth.html" style="width:100%;height:460px;border:none;border-radius:12px;display:block;margin:1.25rem 0 1.5rem;" title="Authorization walkthrough; use arrow keys or Prev/Next to navigate"></iframe>

Add `"auth": "oauth"` to the server entry in `.mcp.json`:

```json
{
  "mcpServers": {
    "gmail": {
      "type": "sse",
      "url": "https://example-mcp-server.googleapis.com/v1/mcp",
      "auth": "oauth",
      "allowedTools": ["list_emails", "read_email"]
    }
  }
}
```

When deepagents detects a server that requires OAuth, it pauses and opens a browser window for the user to log in. Tokens are stored in `~/.deepagents/.state/mcp-tokens/` and reused on subsequent runs.

`allowedTools` lets you restrict which tools from a server the agent can see. This is useful when a server exposes many tools but you only need a subset.

### Human-in-the-Loop (HITL)

Even after authorization, you may want a human to confirm before the agent reads or writes data. Pass `interrupt_on` to `create_deep_agent` with a mapping of tool names to approval settings:

<iframe src="images/m1.l6-mcp-slides-hitl.html" style="width:100%;height:460px;border:none;border-radius:12px;display:block;margin:1.25rem 0 1.5rem;" title="HITL walkthrough; use arrow keys or Prev/Next to navigate"></iframe>

```python
from deepagents import create_deep_agent
from langgraph.checkpoint.memory import MemorySaver

agent = create_deep_agent(
    model=model,
    tools=tools,
    interrupt_on={
        "list_emails": True,
        "read_email":  True,
    },
    checkpointer=MemorySaver(),
)
```

`True` pauses the agent before calling that tool and presents the pending call to the user. The user can approve, edit the inputs, reject, or respond with a message. Execution resumes once a decision is made.

<div style="margin:1.5rem 0;padding:1.25rem 1.5rem;border-radius:12px;border:1px solid #99d3ff;background-color:#e5f4ff;color:#1e4d7a;">
<strong>interrupt_on requires a checkpointer</strong>
<p style="margin:0.5rem 0 0;">When the agent pauses for human approval, it needs to save its current state so execution can resume from exactly the same point after a decision is made. A checkpointer handles that save-and-restore. <code>MemorySaver</code> keeps state in memory and is fine for development; use a persistent checkpointer in production.</p>
</div>

## Lab 4: LangChain Docs (no auth)

The project repo includes `.mcp.json` at the root with the LangChain documentation server pre-configured. You do not need to edit any config for this lab.

Create a new file `python/m1/mcp_agent.py`:

```python
import asyncio
from models import model
from deepagents import create_deep_agent
from deepagents_code.mcp_tools import get_mcp_tools

async def main():
    tools, session_manager, server_infos = await get_mcp_tools(".mcp.json")

    for info in server_infos:
        print(f"{info.name}: {info.status} — {len(info.tools)} tool(s)")

    agent = create_deep_agent(model=model, tools=tools)

    result = await agent.ainvoke({
        "messages": [{"role": "user", "content": "What is the LangGraph interrupt API and when would I use it?"}]
    })
    print(result["messages"][-1].content)

    if session_manager:
        await session_manager.cleanup()

asyncio.run(main())
```

Run it from the repo root:

```bash
uv run ./python/m1/mcp_agent.py
```

Open LangSmith and find the run. You should see the MCP tool call in the trace alongside any other tool calls, listed as a regular tool message in the message history.

<div style="margin:1.5rem 0;padding:1.25rem 1.5rem;border-radius:12px;border:1px solid #99d3ff;background-color:#e5f4ff;color:#1e4d7a;">
<strong>The server_infos check</strong>
<p style="margin:0.5rem 0 0;">Printing <code>server_infos</code> at startup tells you which servers loaded successfully and how many tools each one exposed. If a server has <code>status="error"</code>, its tools will not be available and the agent will not try to call them.</p>

## Lab 5: Gmail with OAuth and HITL

This lab adds a Gmail MCP server to the agent and gates every email access behind a human approval step.

### Step 1: Add Gmail to .mcp.json

Add the Gmail server entry to `.mcp.json` in the project root. Use `allowedTools` to restrict the agent to read-only email access:

```json
{
  "mcpServers": {
    "docs-langchain": {
      "type": "http",
      "url": "https://docs.langchain.com/mcp"
    },
    "gmail": {
      "type": "sse",
      "url": "https://gmail.googleapis.com/v1/mcp",
      "auth": "oauth",
      "allowedTools": ["list_emails", "read_email"]
    }
  }
}
```

<div style="margin:1rem 0;padding:1rem 1.25rem;border-radius:10px;border:1px solid #b3e0b3;background-color:#f0faf0;color:#1a4d1a;font-size:0.9em;">
<strong>Check the current URL</strong><br>
Verify the Gmail MCP server URL from Google's official documentation before running. The course README has the tested URL and any additional setup steps.
</div>

### Step 2: Authenticate

On the first run, deepagents will detect `auth: "oauth"` and open a browser window for you to authorize the Gmail scope. Tokens are saved to `~/.deepagents/.state/mcp-tokens/gmail.json` and reused on subsequent runs.

### Step 3: Add HITL and run

Update `python/m1/mcp_agent.py` to add `interrupt_on` for the Gmail tools:

```python
import asyncio
import uuid
from models import model
from deepagents import create_deep_agent
from deepagents_code.mcp_tools import get_mcp_tools
from langgraph.checkpoint.memory import MemorySaver
from langgraph.types import Command

async def main():
    tools, session_manager, _ = await get_mcp_tools(".mcp.json")

    agent = create_deep_agent(
        model=model,
        tools=tools,
        interrupt_on={
            "list_emails": True,
            "read_email":  True,
        },
        checkpointer=MemorySaver(),
    )

    config = {"configurable": {"thread_id": str(uuid.uuid4())}}

    await agent.ainvoke(
        {"messages": [{"role": "user", "content": "Summarize my three most recent emails."}]},
        config=config,
    )

    state = agent.get_state(config)
    if state.interrupts:
        interrupt = state.interrupts[0].value
        for req in interrupt["action_requests"]:
            print(f"\nPending tool call: {req['name']}")
            print(f"  Input: {req.get('args', {})}")
        decision = input("\nApprove? (y/n): ")
        if decision.strip().lower() == "y":
            result = await agent.ainvoke(
                Command(resume={"decisions": [{"type": "approve"} for _ in interrupt["action_requests"]]}),
                config=config,
            )
            print(result["messages"][-1].content)
        else:
            print("Rejected.")

    if session_manager:
        await session_manager.cleanup()

asyncio.run(main())
```

Run it:

```bash
uv run ./python/m1/mcp_agent.py
```

The agent will call `list_emails`, then pause. You will see the pending tool call printed in the terminal. Type `y` to approve and the agent will continue, fetch the emails, and summarize them.

Check LangSmith to see the full trace: the interrupt, the resume, and the final tool results all appear in the message history.
