# Creating a Deep Agent

## Using a Coding Agent

Before we explore, let's get a bare agent running. Paste the prompt below into
your coding agent (Claude Code, Cursor, etc.) and let it build the file.

```text
Create a minimal LangChain Deep Agent in lca-deepagents/python/m1/scratch_agent.py:

- Import the shared model from python/models.py. Because the script lives in
  python/m1/, add the project root to sys.path first:
      import sys
      from pathlib import Path
      sys.path.insert(0, str(Path(__file__).resolve().parent.parent))
      from models import model
- Create the agent with create_deep_agent(model=model); pass NOTHING else:
  no tools, no system prompt, no filesystem config.
- Invoke it once with the message "What is an LLM?" and print the final
  message's .content.

Also create lca-deepagents/python/.env (next to models.py; you can copy the
provided .env.example). models.py already calls load_dotenv() on this file, so
scratch_agent.py needs no dotenv code of its own. Use these placeholders; I'll
paste my real keys in afterward:

    # LangSmith: tracing & observability (lets you open the run in LangSmith)
    LANGSMITH_API_KEY=lsv2_...
    LANGSMITH_TRACING=true
    LANGSMITH_PROJECT=lca-deepagents

    # Model provider keys: set the one your model in models.py uses
    ANTHROPIC_API_KEY=your-anthropic-api-key   # default: anthropic:claude-haiku-4-5
    OPENAI_API_KEY=your-openai-api-key         # only if you switch to an openai: model

python-dotenv, langchain, langchain-anthropic, and deepagents are already in the
project's pyproject.toml, so no new dependencies are needed.

Run it with (from the lca-deepagents repo root):
  uv run ./python/m1/scratch_agent.py
```

Once your coding agent has created the file, run it:

<RunCode>
:::python
```bash
uv run ./python/m1/scratch_agent.py
```
:::
</RunCode>

<div class="callout">
<strong>Troubleshooting</strong>
<ul>
<li><strong><code>ModuleNotFoundError: No module named 'dotenv'</code></strong> (or any package): you ran the script with <code>python3</code> directly. Use <code>uv run</code> instead; it activates the project's virtual environment where all dependencies live.</li>
<li><strong><code>error: Failed to spawn</code></strong> / <strong><code>Project directory does not exist</code></strong>: you're in the wrong directory. Run the command from the <code>lca-deepagents</code> repo root.</li>
</ul>
</div>

Notice what you *didn't* pass: no tools, no system prompt, no filesystem. Yet
the agent still answers, still knows how to behave, and still has file tools.
Where did all of that come from? That's what the next lessons uncover.
