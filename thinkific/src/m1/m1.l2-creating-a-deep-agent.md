# Creating a Deep Agent

## Using a Coding Agent

Before we explore, let's get a bare agent running. Paste the prompt below into
your coding agent (Claude Code, Cursor, etc.) and let it build the file.

```text
Create python/m1/scratch_agent.py as a minimal deep agent:

import sys
from pathlib import Path
sys.path.insert(0, str(Path(__file__).resolve().parent.parent))
from models import model
from deepagents import create_deep_agent

agent = create_deep_agent(model=model)  # no tools, no system_prompt, no filesystem config
result = agent.invoke({"messages": [{"role": "user", "content": "What is an LLM?"}]})
print(result["messages"][-1].content)

Also copy python/.env.example to python/.env and fill in your API keys.
```

<details>
<summary>Don't have a coding agent? Copy the code directly [click to expand]</summary>
<br>

Create `python/m1/scratch_agent.py` with the following:

```python
import sys
from pathlib import Path
sys.path.insert(0, str(Path(__file__).resolve().parent.parent))
from models import model

from deepagents import create_deep_agent

agent = create_deep_agent(model=model)
result = agent.invoke({"messages": [{"role": "user", "content": "What is an LLM?"}]})
print(result["messages"][-1].content)
```

</details>

Once your coding agent has created the file, run it:

```bash
cd ~/Documents/Github/lca-deepagents && uv run ./python/m1/scratch_agent.py
```

<div class="callout">
<strong>Troubleshooting</strong>
<ul>
<li><strong><code>ModuleNotFoundError: No module named 'dotenv'</code></strong> (or any package): you ran the script with <code>python3</code> directly. Use <code>uv run</code> instead; it activates the project's virtual environment where all dependencies live.</li>
<li><strong><code>error: Failed to spawn</code></strong> / <strong><code>Project directory does not exist</code></strong>: you're in the wrong directory. Run the command from the <code>lca-deepagents</code> repo root.</li>
</ul>
</div>

Notice what you *didn't* pass: no tools, no system prompt, no filesystem. Yet
the agent still answers, still knows how to behave, and still has file tools.
Where did all of that come from? That's what the next lessons uncover.
