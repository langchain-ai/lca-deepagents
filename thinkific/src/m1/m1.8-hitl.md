In the tools lesson, you saw that tools can do more than retrieve information: they can also take actions. In the previous lesson, you saw how messages, threads, and checkpointers let an agent save and resume state. Human-in-the-Loop (HITL) combines those ideas: pause before a sensitive action and let a person approve, edit, or reject it.

The main reason is safety. If an agent is about to send an email, update a database, or delete a file, you may want a human checkpoint before the side effect happens. HITL also lets the agent suspend execution while it waits instead of staying active indefinitely.

## Lab: Reviewing an Email Tool

Open `python/m1/m1.8_hitl.py`. It defines a mock `send_email` tool. The tool does not contact a real email service; it returns a string so the lab is safe to run.

### 1. Define an action tool

```python
from langchain_core.tools import tool

@tool
def send_email(to: str, subject: str, body: str) -> str:
    """Send an email to a recipient."""
    return f"Email sent to {to} with subject {subject!r}. Body: {body}"
```

### 2. Configure HITL on that tool

```python
from deepagents import create_deep_agent
from langgraph.checkpoint.memory import MemorySaver

agent = create_deep_agent(
    model=model,
    tools=[send_email],
    system_prompt=SYSTEM_PROMPT,
    interrupt_on={"send_email": True},
    checkpointer=MemorySaver(),
)
```

`interrupt_on={"send_email": True}` means: if the agent tries to call `send_email`, pause before the tool executes and surface the pending action for review.

The checkpointer is required because the agent needs somewhere to save the paused state. Without it, there is nothing to resume.

### 3. Invoke with a stable thread ID

```python
config = {"configurable": {"thread_id": "m1-7-hitl-demo"}}

result = agent.invoke(
    {
        "messages": [
            {
                "role": "user",
                "content": "Send an email to jane@example.com saying I will be 10 minutes late.",
            }
        ]
    },
    config=config,
)
```

The `thread_id` is how the checkpointer finds this paused run again. When you resume, pass the same `config`.

### 4. Review and resume

```python
from langgraph.types import Command

while result.get("__interrupt__"):
    pending = result["__interrupt__"][0].value
    decisions = []
    for req in pending["action_requests"]:
        print(f"\nApproval required for {req['name']}:")
        print(req["args"])

        choice = input("\nApprove, edit, or reject? (approve/edit/reject): ").strip().lower()
        if choice in ("approve", "yes", "y"):
            decisions.append({"type": "approve"})
        elif choice in ("edit", "e"):
            edited_args = dict(req["args"])
            edited_args["body"] = input("New email body: ")
            decisions.append(
                {
                    "type": "edit",
                    "edited_action": {"name": req["name"], "args": edited_args},
                }
            )
        else:
            decisions.append(
                {"type": "reject", "message": "User rejected this email draft."}
            )

    result = agent.invoke(Command(resume={"decisions": decisions}), config=config)
```

If you approve, `send_email` runs as proposed. If you edit, the tool runs with your revised arguments. If you reject, the tool never runs; the agent receives a cancellation result and should respond accordingly.

### Run it

From the repo root:

```bash
cd python
uv run python m1/m1.8_hitl.py
```

Try all three paths:

- run once and type `approve`;
- run again and type `edit`, then provide a new email body;
- run again and type `reject`.

You should see the pending tool arguments before each decision. That is the safety checkpoint: the side effect is visible before it happens.
