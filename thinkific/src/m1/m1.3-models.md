<strong>What is a provider?</strong>
<p style="margin:0.5rem 0 0;">A provider is a company or platform that hosts AI models and exposes them through an API (e.g., OpenAI, Anthropic, Google). Many providers have a dedicated <code>langchain-&lt;provider&gt;</code> package that implements one or more of LangChain's standard interfaces (chat models, embedding models, vector stores, and more), giving you a consistent API regardless of which provider is underneath. Install the package, pick a model name, and swap providers without changing your code.</p>

The table above covers the main categories used in this course. For the full directory of 50+ supported providers (OpenAI, Anthropic, Google, Groq, Ollama, Mistral, and many more), see the [LangChain chat model integrations](https://docs.langchain.com/oss/python/integrations/chat).

Every lab gets its model from a single shared file, **`python/models.py`**. It exports a `model` object built with `init_chat_model(...)`, and the lab scripts simply do `from models import model`.

To use a different model or provider, edit `models.py` (comment out the active line and uncomment the desired model):

```python
# python/models.py
model = init_chat_model("anthropic:claude-haiku-4-5")   # workshop default, fast & cheap
# model = init_chat_model("ollama:llama3.2")
# model = init_chat_model("openai:gpt-4.1-mini")
# model = init_chat_model("openai:gpt-4.1")
```

If you set up Anthropic or OpenAI in the setup lesson, those work out of the box. If you want to use Azure, Bedrock, or Google, those require an extra install (`uv sync --extra azure|bedrock|google`) and additional env vars; see the notes at the top of `models.py`.

Head to **Lab 1** to run this yourself and see how the output changes across models.

<p>An LLM (Large Language Model) is a type of artificial intelligence system trained on vast amounts of text data to predict and generate human language.</p>
<p><strong>Key characteristics:</strong></p>
<ul>
<li><strong>Scale</strong>: Contains billions to hundreds of billions of parameters (learnable weights), which is why they're "large"</li>
<li><strong>Training</strong>: Learns patterns from massive text datasets using transformer-based neural networks</li>
<li><strong>Capability</strong>: Can perform tasks like answering questions, writing, summarizing, translating, coding, and reasoning</li>
<li><strong>Generative</strong>: Produces text one word (or "token") at a time, based on patterns learned during training</li>
<li><strong>Context-aware</strong>: Maintains understanding of conversation history and context within a session</li>
</ul>
<p><strong>Common examples</strong>: ChatGPT, Claude, Gemini, Llama, and Mistral are all LLMs.</p>
<p>LLMs don't truly "understand" in the human sense; they're sophisticated pattern-matching systems that are remarkably effective at tasks requiring language comprehension and generation.</p>

<div style="margin-top:1rem;">
<p>An <strong>LLM (Large Language Model)</strong> is a type of artificial intelligence model trained on vast amounts of text data to understand and generate human-like language.</p>
<p><strong>Key characteristics:</strong></p>
<ul>
<li><strong>Scale</strong>: Billions to trillions of parameters (e.g., GPT-4, Claude, Llama)</li>
<li><strong>Training</strong>: Self-supervised learning on diverse text corpora (books, websites, code, etc.)</li>
<li><strong>Architecture</strong>: Typically based on the Transformer architecture with attention mechanisms</li>
<li><strong>Capabilities</strong>: Text generation, summarization, translation, coding, reasoning, question answering</li>
</ul>
<p><strong>How it works:</strong> LLMs predict the next token (word/character) in a sequence based on context. Through massive training, they learn patterns, facts, reasoning, and linguistic structures.</p>
<p><strong>Limitations:</strong> They can hallucinate (generate false information), lack true understanding, have knowledge cutoffs, and can be biased by training data.</p>

### Ollama: run a model locally

**[NO API KEY NEEDED]**

Ollama is a tool for downloading and running open-source models directly on your machine. No API key, no data sent to a third party: the model runs entirely on your hardware.

<Video src="https://share.descript.com/view/5t0FLKmmD7h" />

**Step 1: Install Ollama** from [ollama.com](https://ollama.com)

**Step 2: Pull a model:**

```bash
ollama pull llama3.2
```

**Step 3: Update `python/models.py`.** Comment out the active line and uncomment the Ollama line:

```python
# model = init_chat_model("anthropic:claude-haiku-4-5")
model = init_chat_model("ollama:llama3.2")
```

**Step 4: Run the lab as normal.** No other code changes needed.

The agent code and system prompt are identical. Only the model line in `models.py` is different.

> **Local only.** Ollama also offers hosted cloud models at ollama.com, which do require an API key. These steps are for running a model locally on your own machine.

## Recap

- `init_chat_model("provider:model")` targets any LangChain-supported backend with a single string; swap the prefix to switch providers, swap the suffix to switch models
- All labs share one model object from `python/models.py`; changing that one line is the only code change needed to run a different model
- Three categories in this course: Proprietary & Conversational, Advanced Reasoning, and Open-Source/Open-Weight; 50+ providers are supported in total
- Swapping the model changes nothing else in the agent; the system prompt, tools, and behavior instructions stay identical
