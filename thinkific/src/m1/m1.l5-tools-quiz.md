<MCQ
    question="Why does Human-in-the-Loop require a checkpointer?"
    choices='["To save graph state so execution can pause and resume from the same point after a human decision", "To log tool call arguments to a file for audit purposes", "To validate that the human's approval matches the tool schema", "To prevent the LLM from calling the tool more than once"]'
    correctIndex={0}
    explanation="When interrupt() fires, the graph suspends and must save its current state so it can resume from exactly the same point once the human responds. A checkpointer handles that save-and-restore. Without one, the graph has no way to pause mid-run."
/>
