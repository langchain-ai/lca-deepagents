The first two modules covered what a deep agent does and where it runs. This module turns to a different question: what does the agent know, and for how long?

**Message History & Summarization** addresses the context window. A session can grow long as tools return output and the conversation branches. SummarizationMiddleware monitors token usage and compacts old messages into a summary before the context limit is reached, so the agent keeps running without developer intervention.

**Skills** address domain knowledge. Some instructions are always relevant; others belong in the prompt only when a particular task calls for them. A skill is a SKILL.md file whose name and description stay in context permanently, while the full instructions load on demand when a task matches. Skills follow an open standard and travel between agents and teams.

**Memory** extends what the agent knows across sessions. A plain agent forgets everything when a conversation ends. MemoryMiddleware injects markdown memory files into the system prompt on every call, carrying user preferences, project conventions, and accumulated context forward from one session to the next.
