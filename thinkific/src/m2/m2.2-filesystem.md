## The filesystem backends

You can select different backends depending on your agent's storage needs. The list below describes the built-in options. You can also build a custom backend for your needs (GitHub, S3, etc.).

**State**: the default. Files live in state, also referred to as "checkpoint or short-term memory". State is thread-scoped and not visible between threads.

**Filesystem**: reads and writes files on the local disk. Scoped to a `root_dir` you specify. 

**Store**: backed by the store, a built-in persistent key/value store, referenced with a namespace. It is a cross-thread store, visible to all threads.

**Context Hub**: LangSmith managed context storage. Useful for reference material, Agent.md, skills and more. Learn more about the LangSmith Context Hub [here](https://docs.langchain.com/langsmith/context-hub).

**Composite**: routes different path prefixes to different backends. Lets you mix different backends in a single agent (e.g. `/memories/` → Store, `/skills/` → Store, everything else → State).

**Custom**: implement the backend interface yourself to plug in any storage system.

The simplest setup is just to pass one backend directly to `create_deep_agent`. The example here uses `StoreBackend`, which persists files across threads. The `namespace` argument is a callable that returns a tuple — it groups all files written by this agent under a single key in the store.

[Scroll to Composite backend]

Composite is for when different files have different requirements. You define routes by path prefix and anything that doesn't match a route falls through to the default. Here `/memories/` and `/skills/` both go to the durable store, while everything else stays ephemeral in State. The agent never sees any of this - it just calls `read_file` and the routing happens transparently.

[Scroll to Permissions]

Each rule has three fields: which operations it covers, which paths it applies to using glob patterns, and the mode. The default mode is allow, but you can also deny outright or use interrupt - which pauses for human approval instead of blocking silently, the same pattern we saw with HITL.

## Single backend

Pass any backend directly to `create_deep_agent`. Here the agent stores all files in the LangGraph store, persisted across threads.

```python {10,12}
# python/m2/m2.2_agent.py
from models import model
from deepagents import create_deep_agent
from deepagents.backends import StoreBackend
from langgraph.store.memory import InMemoryStore

agent = create_deep_agent(
    model=model,
    backend=StoreBackend(namespace=lambda _: ("files",)),
    store=InMemoryStore(),
)
```

The namespace callable returns a tuple that groups all files written by this agent under `("files",)` in the key-value store.

## Composite backend

Most agents have files with different lifetimes. Scratch files written during a task only need to survive the current thread. Memories and skills need to persist across threads and sessions. Using a single backend forces a trade-off: either everything is ephemeral or everything is durable. `CompositeBackend` resolves this by routing path prefixes to different backends. Everything not matched by a route falls through to the `default`.

```python {12,13,16}
# python/m2/m2.2_agent.py
from models import model
from deepagents import create_deep_agent
from deepagents.backends import CompositeBackend, StateBackend, StoreBackend
from langgraph.store.memory import InMemoryStore

agent = create_deep_agent(
    model=model,
    backend=CompositeBackend(
        default=StateBackend(),
        routes={
            "/memories/": StoreBackend(namespace=lambda _: ("files",)),
            "/skills/": StoreBackend(namespace=lambda _: ("files",)),
        },
    ),
    store=InMemoryStore(),
)
```

- `/memories/` → durable store, persists across threads
- `/skills/` → durable store, persists across threads
- Everything else → `StateBackend`, thread-scoped and ephemeral

## Permissions

Permissions are enforced in code, not in the prompt. The model cannot bypass them. Pass a list of `FilesystemPermission` rules to `create_deep_agent`; rules are evaluated in order and the first match wins. If nothing matches, the operation is allowed.

Each rule has three fields:

| Field | Values | Default | Description |
|---|---|---|---|
| `operations` | `"read"`, `"write"` | - | `"read"` covers `ls`, `read_file`, `glob`, `grep`. `"write"` covers `write_file`, `edit_file`. |
| `paths` | glob patterns | - | e.g. `["/memories/**"]`. Supports `**` and `{a,b}` alternation. |
| `mode` | `"allow"`, `"deny"`, `"interrupt"` | `"allow"` | `"interrupt"` pauses for human approval instead of blocking. |

```python {17-23}
# python/m2/m2.2_agent.py
from models import model
from deepagents import create_deep_agent, FilesystemPermission
from deepagents.backends import CompositeBackend, StateBackend, StoreBackend
from langgraph.store.memory import InMemoryStore

agent = create_deep_agent(
    model=model,
    backend=CompositeBackend(
        default=StateBackend(),
        routes={
            "/memories/": StoreBackend(namespace=lambda _: ("files",)),
            "/skills/": StoreBackend(namespace=lambda _: ("files",)),
        },
    ),
    store=InMemoryStore(),
    permissions=[
        FilesystemPermission(
            operations=["write"],
            paths=["/skills/**"],
            mode="deny",
        ),
    ],
)
```

The agent can read `/skills/` but not write to it; skills are managed by application code, not the agent. `/memories/` remains fully read/write.

## Recap

- All backends expose the same tool surface
- **State** is the default: fast and zero-config, but thread-scoped
- **Store** is the right choice when files need to survive across sessions
- **Composite** lets you combine backends by path prefix

store.put(

description: Act as a Chinook sales rep; answer questions about customers, invoices, and the music catalog.
---

You are a sales representative for Chinook Digital Music Store.

**Your responsibilities:**
- Look up customer accounts and purchase history
- Recommend music based on a customer's genre and artist preferences
- Answer questions about the catalog (artists, albums, tracks, pricing)
- Summarize invoice history for a customer

    model=model,

        default=StateBackend(),

<script>
(function () {
  var tabs = document.querySelectorAll('.lt-tab');
  function show(p) {
    tabs.forEach(function (t) {
      var on = t.getAttribute('data-p') === p;
      t.classList.toggle('active', on);
      t.setAttribute('aria-selected', on ? 'true' : 'false');
    });
    document.querySelectorAll('.lt-panel').forEach(function (panel) {
      panel.classList.toggle('active', panel.id === 'p-' + p);
    });
  }
  tabs.forEach(function (t) {
    t.addEventListener('click', function () { show(t.getAttribute('data-p')); });
  });
})();
</script>
