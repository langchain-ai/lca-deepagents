The filesystem backends control **where files are stored**. Each backend implements the same interface (`ls`, `read_file`, `write_file`, `edit_file`, `glob`, `grep`), so the agent code stays the same regardless of which one you choose.

## The backends

**State**: the default. Files live in LangGraph state, also referred to as "checkpoint or short-term memory". It is scoped to the current thread. Persists as long as the thread persists.

**Filesystem**: reads and writes files on the local disk. Scoped to a `root_dir` you specify. 

**Store**: backed by the store. In production, this is a PostgreSQL database; locally, it can be an in-memory database or a local database. Files persist across threads. 

**Context Hub**: files are injected directly into the model's context window, without an explicit read call. Useful for reference material, the agent should always have in view. Learn more about the LangSmith Context Hub [here](https://docs.langchain.com/langsmith/context-hub).

**Composite**: routes different path prefixes to different backends. Lets you mix different backends in a single agent (e.g. `/memories/` → Store, `/skills/` → Store, everything else → State).

**Custom**: implement the backend interface yourself to plug in any storage system.

---

## Single backend

Pass any backend directly to `create_deep_agent`. Here the agent stores all files in the LangGraph store, persisted across threads.

<Tabs>
<Tab title="Python">

```python {10,12}
# python/m2/agent.py
from models import model
from deepagents import create_deep_agent
from deepagents.backends import StoreBackend
from langgraph.store.memory import InMemoryStore

agent = create_deep_agent(
    model=model,
    backend=StoreBackend(
        namespace=lambda rt: (rt.server_info.user.identity,),
    ),
    store=InMemoryStore(),  # swap for PostgresStore in production
)
```

</Tab>
</Tabs>

`rt` is the LangGraph runtime passed to the backend at invocation time. `rt.server_info.user.identity` is the caller's user ID; using it as the namespace key keeps each user's files isolated in the store.

## Composite backend

`CompositeBackend` routes path prefixes to different backends. Everything not matched by a route falls through to the `default`.

<Tabs>
<Tab title="Python">

```python {12,13,16}
# python/m2/agent.py
from models import model
from deepagents import create_deep_agent
from deepagents.backends import CompositeBackend, StateBackend, StoreBackend
from langgraph.store.memory import InMemoryStore

agent = create_deep_agent(
    model=model,
    backend=CompositeBackend(
        default=StateBackend(),
        routes={
            "/memories/": StoreBackend(namespace=lambda rt: (rt.server_info.user.identity,)),
            "/skills/": StoreBackend(namespace=lambda rt: (rt.server_info.user.identity,)),
        },
    ),
    store=InMemoryStore(),  # swap for PostgresStore in production
)
```

</Tab>
</Tabs>

- `/memories/` → durable store, persists across threads
- `/skills/` → durable store, persists across threads
- Everything else → `StateBackend`, thread-scoped and ephemeral

## Permissions

Pass a list of `FilesystemPermission` rules to `create_deep_agent` to control which paths the agent can read or write. Rules are evaluated in order; the first match wins. If nothing matches, the operation is allowed.

Each rule has three fields:

| Field | Values | Default | Description |
|---|---|---|---|
| `operations` | `"read"`, `"write"` | - | `"read"` covers `ls`, `read_file`, `glob`, `grep`. `"write"` covers `write_file`, `edit_file`. |
| `paths` | glob patterns | - | e.g. `["/memories/**"]`. Supports `**` and `{a,b}` alternation. |
| `mode` | `"allow"`, `"deny"`, `"interrupt"` | `"allow"` | `"interrupt"` pauses for human approval instead of blocking. |

<Tabs>
<Tab title="Python">

```python {17-23}
# python/m2/agent.py
from models import model
from deepagents import create_deep_agent, FilesystemPermission
from deepagents.backends import CompositeBackend, StateBackend, StoreBackend
from langgraph.store.memory import InMemoryStore

agent = create_deep_agent(
    model=model,
    backend=CompositeBackend(
        default=StateBackend(),
        routes={
            "/memories/": StoreBackend(namespace=lambda rt: (rt.server_info.user.identity,)),
            "/skills/": StoreBackend(namespace=lambda rt: (rt.server_info.user.identity,)),
        },
    ),
    store=InMemoryStore(),
    permissions=[
        FilesystemPermission(
            operations=["write"],
            paths=["/skills/**"],
            mode="deny",
        ),
    ],
)
```

</Tab>
</Tabs>

The agent can read `/skills/` but not write to it; skills are managed by application code, not the agent. `/memories/` remains fully read/write.

---

## Recap

- All backends expose the same tool surface; the agent never knows which one it's talking to
- **State** is the default: fast and zero-config, but thread-scoped
- **Store** is the right choice when files need to survive across sessions
- **Composite** lets you combine backends by path prefix

## Lab 1: Permissions

The lab agent uses a `CompositeBackend` with `/memories/` and `/skills/` both routed to the store. A `FilesystemPermission` rule denies all writes to `/skills/**`.

The agent is pre-seeded with a Chinook sales skill and asked to read it, then add a promotion note. You'll see it read successfully and get blocked on the write.

```python
# python/m2/agent.py
import sys
from pathlib import Path
sys.path.insert(0, str(Path(__file__).resolve().parent.parent))
from models import model
from deepagents import create_deep_agent, FilesystemPermission
from deepagents.backends import CompositeBackend, StateBackend, StoreBackend
from deepagents.backends.utils import create_file_data
from langgraph.store.memory import InMemoryStore

store = InMemoryStore()

# Seed the Chinook sales skill
store.put(
    ("student",),
    "/skills/chinook-sales/SKILL.md",
    create_file_data("""\
---
name: chinook-sales
description: Act as a Chinook sales rep; answer questions about customers, invoices, and the music catalog.
---

You are a sales representative for Chinook Digital Music Store.

**Your responsibilities:**
- Look up customer accounts and purchase history
- Recommend music based on a customer's genre and artist preferences
- Answer questions about the catalog (artists, albums, tracks, pricing)
- Summarize invoice history for a customer

**Chinook schema (key tables):**
- `Customer`: CustomerId, FirstName, LastName, Email, SupportRepId
- `Employee`: EmployeeId, Title (e.g. "Sales Support Agent")
- `Invoice`: InvoiceId, CustomerId, InvoiceDate, Total
- `InvoiceLine`: InvoiceId, TrackId, UnitPrice, Quantity
- `Track`: TrackId, Name, AlbumId, GenreId, UnitPrice
- `Album`: AlbumId, Title, ArtistId
- `Artist`: ArtistId, Name
- `Genre`: GenreId, Name
"""),
)

agent = create_deep_agent(
    model=model,
    backend=CompositeBackend(
        default=StateBackend(),
        routes={
            "/memories/": StoreBackend(namespace=lambda _rt: ("student",)),
            "/skills/": StoreBackend(namespace=lambda _rt: ("student",)),
        },
    ),
    store=store,
    permissions=[
        FilesystemPermission(
            operations=["write"],
            paths=["/skills/**"],
            mode="deny",
        ),
    ],
)

result = agent.invoke(
    {
        "messages": [
            {
                "role": "user",
                "content": (
                    "Read the chinook-sales skill file, then add this note to it: "
                    "'Current promotion: 20% off all Jazz albums through end of month.'"
                ),
            }
        ]
    },
    config={"configurable": {"thread_id": "lab-m2.2"}},
)

print(result["messages"][-1].content)
```
