## The Agent's Filesystem

Every deep agent has a backend: a storage layer that determines where files live when the agent reads and writes them. Two backends matter here:

| Backend | Where files live | Persists to disk? |
|---|---|---|
| `StateBackend` | In-memory | No; gone when the run ends |
| `FilesystemBackend` | A directory on your machine | Yes |

`StateBackend` is the default. The agent can use all its file tools, but nothing is written to disk. Switch to `FilesystemBackend` when you want the agent's work to persist, be inspectable, or be shared with other processes.

### Configuring FilesystemBackend

Pass it via the `backend` parameter in `create_deep_agent`:

```python
from deepagents.backends import FilesystemBackend

agent = create_deep_agent(
    model=model,
    backend=FilesystemBackend(root_dir="workspace/"),
)
```

`root_dir` sets the working directory for all file operations. Relative paths the agent uses are resolved under this directory. Set `virtual_mode=True` to prevent the agent from escaping the root via `..` or absolute paths.

### The file tools

These tools are always present, regardless of backend. The backend determines only where they read and write:

| Tool | What it does |
|---|---|
| `ls` | List directory contents |
| `read_file` | Read a file, with optional line offset and limit |
| `write_file` | Create a new file (fails if it already exists) |
| `edit_file` | Replace a string inside an existing file |
| `glob` | Find files by name pattern, including `**` for recursion |
| `grep` | Search file contents for a literal string |

The agent decides which tools to call and in what order. A typical write-then-read sequence: `write_file` to create a draft, `read_file` to check it, `edit_file` to revise a section.

### System directories

Two directories in the workspace are managed automatically by the SDK:

- `/conversation_history/`: when the context window fills up, older messages are compressed and archived here
- `/large_tool_results/`: when a single tool result is too large to fit inline, it is evicted here and replaced with a file reference

You do not need to manage these. They are covered in detail in Module 3.

## Lab 5: Filesystem Agent

You will build an agent backed by a real on-disk workspace and watch it use the file tools to plan and write a structured document. After the run, the files will be on disk and you can open them directly.

---

### Setup

Create the workspace directory the agent will write into:

```bash
mkdir -p ~/Documents/Github/lca-deepagents/python/m2/workspace
```

---

### 1. Create `filesystem_agent.py`

Create `python/m2/filesystem_agent.py`:

```python
import sys
from pathlib import Path
from deepagents.backends import FilesystemBackend
from deepagents import create_deep_agent
sys.path.insert(0, str(Path(__file__).resolve().parent.parent))
from models import model

WORKSPACE = Path(__file__).parent / "workspace"
WORKSPACE.mkdir(exist_ok=True)

agent = create_deep_agent(
    model=model,
    backend=FilesystemBackend(root_dir=str(WORKSPACE), virtual_mode=True),
    system_prompt=(
        "You are a planning assistant. When given a project, create a structured plan "
        "and save it to a markdown file. Use clear headings and a task checklist."
    ),
)

result = agent.invoke({"messages": [{"role": "user", "content": "Plan a 4-week onboarding programme for a new sales hire. Save it to onboarding-plan.md"}]})
print(result["messages"][-1].content)
```

Before running, try swapping in a different task:

- `"Plan a product launch for a new mobile app. Save it to launch-plan.md"`
- `"Create a weekly meal prep plan for a vegetarian. Save it to meal-plan.md"`
- `"Outline a 30-day fitness programme for a beginner. Save it to fitness-plan.md"`

---

### Run it

```bash
cd ~/Documents/Github/lca-deepagents
```

```bash
uv run ./python/m2/filesystem_agent.py
```

---

### What to check

Open `python/m2/workspace/` in your file explorer or editor. The plan file the agent created should be there. If you used the default prompt, look for `onboarding-plan.md`.

In **LangSmith**, open the run and look for the `write_file` call in the tool sequence. You will see the exact content the agent wrote, the path it chose, and any `read_file` or `edit_file` calls it made to revise the output.

<div class="callout">
<strong>StateBackend vs FilesystemBackend</strong><br>
Remove the <code>backend=</code> argument from <code>create_deep_agent</code> and run again. The agent will still use file tools and complete the task, but nothing will appear on disk. That is <code>StateBackend</code>: the same interface, in-memory only. Swapping the backend is the only change needed to make the agent's work persistent.
